# Product

## Register

product

## Users

1Exchange is for a single local operator who needs to inspect exchange credentials, account assets, positions, and product coverage from one machine. They are usually debugging adapters, checking account exposure, or validating that a credential can read data correctly.

## Product Purpose

The product provides a local, read-first control surface for multiple exchange adapters. Success means the user can see service health, supported exchanges, saved credentials, positions, and products without reading raw API output or exposing credential payloads.

## Brand Personality

Calm, precise, operational. The interface should feel like a trustworthy trading-console tool, not a marketing dashboard.

## Anti-references

Avoid crypto hype visuals, neon finance cliches, decorative charts without data, oversized hero sections, glass panels, and generic SaaS landing-page patterns.

## Design Principles

1. Show live system state first.
2. Keep credentials private and only show metadata.
3. Make exchange and credential context explicit before showing account data.
4. Prefer dense readable tables over decorative cards.
5. Treat empty states as instructions for the next useful action.

## Accessibility & Inclusion

Target WCAG AA contrast, keyboard-accessible navigation and forms, visible focus states, readable table layouts, and reduced-motion-safe transitions.
